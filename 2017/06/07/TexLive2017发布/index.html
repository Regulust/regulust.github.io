TeX Live 2017 发布

2017-06-07
  • News
  • 技术

TeX Live 2017 正式版已经于6月4日发布，各镜像站差不多已经同步完成，老版本需要重新安装（新旧可并存）。
下载说明地址：http://www.tug.org/texlive/acquire.html
中文版说明文档：http://www.tug.org/texlive/doc/texlive-zh-cn/texlive-zh-cn.pdf
推荐使用”Downloading one huge ISO file (typically to burn your own DVD).”的方式下载完整镜像文件安装的方式。

TeX Live 2017更新说明节取：
LuaTEX: 更多 callback，更多排版控制，更多对内部机制的控制；在⼀些平台增加了⽤来动态载⼊
代码的 ffi 库。
pdfTEX: 去年的 SOURCE_DATE_EPOCH_TEX_PRIMITIVES 环境变量被改名为 FORCE_SOURCE_DATE，
没有功能改变。如果 \pdfpageattr token 列表包含了字符串 /MediaBox，则会忽略输出默认的
/MediaBox。
XeTEX: Unicode/OpenType 数学⽀持现在是基于 HarfBuzz 的 MATH table ⽀持实现的；还有⼀
些 bug 修正。
11 翻译说明 42
Dvips: 保证最后⼀次设置的 papersize special 是最后选中的纸张尺⼨，和 dvipdfmx 以及宏包希
望的效果⼀致；-L0 选项 (L0 配置⽂件设置) 可以⽤来恢复以前的⾏为，也就是第⼀次设置纸张尺⼨的
special 才有效。
epTEX, eupTEX: 来 ⾃ pdfTEX 新 的 原 ⽣ 命 令 \pdfuniformdeviate, \pdfnormaldeviate,
\pdfrandomseed, \pdfsetrandomseed, \pdfelapsedtime, \pdfresettimer。
MacTEX: 从这⼀年开始，MacTEX 只⽀持 Apple 还在发布安全更新的 Mac OS X 版本，使⽤的平台
名称是 x86_64-darwin，⽬前这包括 Yosemite, El Capitan, 和 Sierra (10.10 及以后的)。给更⽼版本的
Mac OS X ⽤的⼆进制程序不会包含在 MacTEX ⾥⾯，但 TEX Live ⾥还会提供 (x86_64-darwinlegacy,
i386-darwin, powerpc-darwin)。
基础架构: TEXMFLOCAL 树现在 (默认) 是在 TEXMFSYSCONFIG 和 TEXMFSYSVAR 之前搜索了；这是
为了更好地符合本地⽂件优先于系统⽂件的原则。另外，tlmgr 有⼀个新的 shell 模式，进⾏交互和
脚本使⽤，和⼀个新的 conf auxtrees 功能来⽅便添加和删除额外的⽬录树。
updmap 和 fmtutil: 这些脚本如果在没有明确指定是系统模式 (updmap-sys, fmtutil-sys, 或者
-sys 选项)，还是⽤户模式 (updmap-user, fmtutil-user, 或者 -user 选项) 的时候会给出警告。这是为
了减少常见的不⼩⼼启⽤⽤户模式⽽导致系统更新⽆法⽣效。细节请参见 http://tug.org/texlive/
scripts-sys-user.html。
install-tl: Mac 下的个⼈路径现在默认设置为和 MacTEX ⼀样的 (~/Library/…)。增加了新
选项 -init-from-profile 来以特定 profile 启动安装；新命令 P 来直接保存⼀个 profile；新的 profile
变量名 (但先前的名字也还⽀持)。
SyncTEX: 临时⽂件现在被改为了 foo.synctex(busy)，⽽不是 foo.synctex.gz(busy) (不再
有 .gz)。需要删除临时⽂件的前端和编译系统因此需要调整。
其他⼯具: texosquery-jre8 是⼀个新的跨平台程序，⽤来在 TEX ⽂档中获取系统 locale 和其他
OS 信息；它已经包含在了 shell_escape_commands 中，可以在受限的 shell 环境中执⾏。(texosquery
也⽀持更⽼的 JRE 版本，但不能在受限模式下启⽤，因为 Oracle 由于安全问题不再⽀持那些版本了。)
平台：见上述的 MacTEX 条⽬。没有其他的变化。
